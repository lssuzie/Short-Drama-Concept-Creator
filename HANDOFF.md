## 待实现：流水线重设计

### 核心思路（已确认）
三步流程：`① 生成 Prompt → ② 粘贴结果 → ③ 一键「入库」`

- 「入库」是唯一保存动作，质检和反馈变成入库后的附属
- 粘贴区只保留一个「📚 入库」主按钮
- 去掉「保存到云端」「自动质检」「写反馈」三个独立按钮
- 点「入库」→ 自动跑质检 → 显示结果 → 弹出反馈区 → 保存到创意库

### 创意库改造
- 「历史记录」→「创意库」（HTML 已改，JS 待改）
- 每张卡片显示：概念标题（从《》提取）+ 结果摘要（前80字）+ 标签 + 日期
- 每张卡片右上角加 × 逐条删除按钮
- `loadHist()` 已改为加载结果到粘贴区（不加载 prompt）

### 删除的代码（已清理）
- 输出面板的「保存到云端」按钮已移除
- `saveHist()` 不再接收 prompt 参数
- `gen()` 不再调用 `saveHist()`
- `autoSaveResult()` 只保存草稿到 localStorage
